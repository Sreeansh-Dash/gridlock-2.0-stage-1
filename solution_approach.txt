GRIDLOCK 2.0 - Traffic Demand Prediction
Solution Overview and Approach

This document outlines the end-to-end machine learning solution designed to predict traffic demand. The primary approach uses a robust ensemble of gradient boosting algorithms: LightGBM, XGBoost, and CatBoost.
________________________________________________________________________________________________________________________________________________________________________________

1. Approach
The methodology follows a structured machine learning pipeline:
	Data Loading & EDA: Reading train/test datasets and analyzing basic statistics, distributions, and null values.
	Feature Engineering: Extracting complex spatial, temporal, and interaction features from raw data to give models more predictive power.
	Model Training & Validation (Two-Stage Pipeline): 
	    Stage 1 (Validation): Using a strict 80/20 Time-Based Holdout to provide a realistic local validation score without future leakage. Target aggregations are computed using a strict 5-Fold Out-Of-Fold (OOF) method on the training data to completely eliminate target leakage.
	    Stage 2 (Final Retraining): After local validation, the models are retrained on 100% of the training data to ensure they learn from the most recent trends right before the test cutoff.
	Hyperparameter Tuning: Models use finely tuned hyperparameters optimized for deeper, structured trees, balancing complexity with robust OOF aggregation features.
	Ensemble Strategy: A weighted blending of predictions from LightGBM, XGBoost, and CatBoost.
	Post-processing: The final blended predictions are clipped to fall within the observed min/max range of the training targets to prevent anomalous extreme predictions.

2. Feature Engineering Details
To capture the complex patterns in traffic demand, extensive feature engineering was applied:

    Geospatial Features:
    	Decoding: Geohashes were decoded into exact `lat` and `lon` coordinates using `pygeohash`.
    	Hierarchical Grouping: Geohash prefixes of length 4 and 5 were extracted and label-encoded to capture macro-level spatial trends (grouping nearby areas).

    Temporal Features:
    	Time Extraction: Parsed `timestamp` into `hour`, `minute`, and calculated `time_minutes` (total minutes since midnight).
        Binning: Created `is_rush_hour` (7-9 AM, 5-8 PM) and `time_of_day_bin` (night, morning, afternoon, evening, late_night).
        Cyclical Encoding: Applied sine and cosine transformations to `hour`, `minute`, and `day` to help the models understand the cyclical nature of time (e.g., 23:59 is 		     		   close to 00:00).
    	Day Features: Created a pseudo day-of-week (`day_mod7`) and `is_weekend` flags.

    Categorical Encoding:
        Label Encoding was applied to string variables: `RoadType`, `Weather`, `LargeVehicles`, and `Landmarks`.

    Interaction Features:
        Created cross-features to capture synergies, such as `hour_road` (Hour × RoadType), `weather_largeveh` (Weather × LargeVehicles), `temp_road` (Temperature × RoadType), 	`lanes_road` (Lanes × RoadType), and `landmarks_road` (Landmarks × RoadType).
        Temperature was also grouped into logical bins (cold, cool, mild, hot).

    Aggregate / Statistical Features:
        Calculated target statistics based on the training data. This includes the mean, median, standard deviation, and count of traffic demand grouped by `geohash`, `hour`, 		`RoadType`, and the combination of `geohash` and `hour`. These aggregates provide a historical baseline demand for specific locations and times.

3. Tools and Technologies Used
	Python: The core programming language used for the entire pipeline.
	Pandas & NumPy: Used for robust data manipulation, aggregation, and mathematical operations.
	Pygeohash: For decoding geohash strings into geographic coordinates.
	Scikit-Learn: For Label Encoding, K-Fold cross-validation splitting, and evaluation metrics (R² score).
	LightGBM, XGBoost, CatBoost: The core gradient boosting frameworks used for regression modeling.
	Optuna: An automatic hyperparameter optimization framework used to fine-tune the LightGBM model.
	AI Agent (Gemini 3.1 Pro): Used as an advanced pair-programming assistant.

4. Relevant Source Files
The project directory (GRIDLOCK2.0`) contains the following key files:
   `traffic_demand_solution.py`: The primary Python script containing the full end-to-end pipeline (EDA, Feature Engineering, Training, Tuning, Ensemble, and Inference).
   `traffic_demand_solution.ipynb`: A Jupyter Notebook equivalent of the solution, typically used for cell-by-cell execution, interactive debugging, and visualization.
   `create_notebook.py`: A utility script to generate or convert the Python script into a Jupyter Notebook.
   `dataset/` (Directory): Contains the core data files:
    *   `train.csv`: The historical training data containing features and the target `demand`.
    *   `test.csv`: The inference data used to generate final predictions.
    *   `sample_submission.csv`: The required format for the final output.
   `submission.csv`: The final output file containing the optimized ensemble predictions ready for submission.
